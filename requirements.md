## 1. 概要

本アプリは汎用会計ソフトではなく、**フリーランスWebコーダー / フロントエンドエンジニア本人が毎日開く経営ダッシュボード**である。売上・利益・案件・固定費・自己投資・キャッシュフロー・税金対策を1つのアプリで完結させ、経営判断を最短で行えることを目的とする。

**プロダクト原則**（すべての要件が満たすこと）

1. フリーランスWebコーダーの経営判断に直接役立つこと
2. 入力の手間を増やさないこと（自動計算・自動補完を最大化）
3. 保守性を損なわないこと（疎結合・責務分離）
4. 既存の設計思想と整合すること

---

## 2. スコープ

本要件で対象とする機能領域は以下。

- 入金管理（請求・入金の追跡）
- 税金積立シミュレーター
- キャッシュフロー管理
- ダッシュボードカスタマイズ

既存機能（案件管理・固定費・サブスク・自己投資・売上/利益集計）は前提として存在し、本書は主に上記4領域と既存機能への影響を定義する。

---

## 3. 用語定義

| 用語 | 定義 |
|---|---|
| 売上（請求ベース） | 請求金額の合計。案件が請求段階に達した時点で計上 |
| 入金（入金ベース） | 実際に入金された金額。キャッシュフローで使用 |
| 未入金 | ステータスが「入金済み」以外の請求 |
| 遅延 | 入金予定日を過ぎても未入金の請求 |
| 予測残高 | 現在残高 ＋ 来月入金予定 − 来月固定費予定 |

## 4. 機能要件

### 4.1 入金管理

**FR-101 請求レコードの管理**
案件ごとに請求・入金レコードを作成・編集・削除できること。
管理項目：請求日 / 入金予定日 / 入金日 / 請求金額 / 入金金額 / 請求書番号 / メモ。
- AC: 案件に紐づく請求を新規作成できる。
- AC: 金額は円単位の整数で保存される。

**FR-102 ステータス管理**
各請求は次のステータスを持つ：`未請求` / `請求済み` / `入金待ち` / `入金済み`。
- AC: ステータスを任意に変更でき、一覧でステータス別に絞り込める。

**FR-103 入金記録のワンクリック化**
「入金済みにする」操作で、入金日＝当日・入金金額＝請求金額を自動入力できること。
- AC: ワンクリックで `入金済み` に遷移し、差額があるときのみ手修正できる。

**FR-104 入力補完**
案件を選択すると請求金額を案件の契約金額から自動補完すること。
- AC: 案件選択時に請求金額欄が自動で埋まる（上書き可能）。

**FR-105 入金サマリーの表示**
ダッシュボードに以下を表示すること。
- 未入金総額（未入金の請求金額合計）
- 今月入金予定（入金予定日が当月かつ未入金の合計）
- 入金遅延案件（一覧・件数・総額）
- AC: 3指標が正しい集計値で表示される。

**FR-106 遅延の可視化**
入金予定日を過ぎて未入金の請求を、警告色バッジ付きで通知対象として上部に表示すること。
- AC: `入金予定日 < 当日 かつ 未入金` の請求が遅延として抽出される。
- AC: 遅延判定は保存カラムではなく都度算出で行う。

---

### 4.2 税金積立シミュレーター

**FR-201 納税額の概算試算**
年間利益を基に、所得税・住民税・個人事業税・国民健康保険・消費税（有効時）の推定納税額を概算表示すること。
- AC: 対象税目の概算合計が推定納税額として算出される。
- AC: 消費税は設定で有効化した場合のみ計算対象に含む。

**FR-202 月次積立推奨額の表示**
推定納税額を基に、毎月積み立てるべき推奨金額を表示すること。
- AC: 推奨額＝推定納税額 ÷ 残り月数（または12）で算出される。

**FR-203 ダッシュボード表示**
現在の利益・推定納税額・毎月の推奨積立額をダッシュボードに表示すること。

**FR-204 概算である旨の明示**
試算値は概算・参考値であり、正確な税額は確定申告で確定する旨を画面上に必ず明記すること。
- AC: 税金積立の画面/カードに免責の注記が常時表示される。

**FR-205 税率・計算式の可変化**
税率・控除・計算パラメータを設定画面から変更できること。
- AC: 設定変更後、試算結果に即時反映される。

---

### 4.3 キャッシュフロー管理

**FR-301 資金状況の表示**
以下を表示すること：現在の口座残高 / 今月の収入 / 今月の支出 / 来月の固定費予定 / 来月の入金予定 / 予測残高。
- AC: 予測残高＝現在残高＋来月入金予定−来月固定費予定 で算出される。
- AC: 収入は入金ベース、支出は固定費・サブスク・経費から集計される。

**FR-302 収支推移グラフ**
月次の収支推移（収入・支出・残高）をグラフ表示すること。
- AC: 過去数ヶ月の推移が折れ線/棒グラフで確認できる。

---

### 4.4 ダッシュボードカスタマイズ

**FR-401 表示カードの選択**
ホーム画面に表示するカードを候補から選び、表示/非表示を切り替えられること。
表示候補：今月売上 / 今月利益 / 年間利益 / 案件ランキング / 利益率ランキング / 固定費 / サブスク / 未入金 / 自己投資額 / 売上目標 / 税金積立 / キャッシュフロー。
- AC: 各カードの表示/非表示を個別に切り替えられる。

**FR-402 並び替え**
カードをドラッグ＆ドロップで並び替えできること。
- AC: 並び順が保存され、再訪時に維持される。

**FR-403 設定の永続化**
表示/非表示と並び順を永続化すること。
- AC: リロード後も設定が保持される。

## 5. データ要件

**DR-01 金額の保持形式**
すべての金額は円単位の整数で保持する。表示時のみ3桁区切りに整形する。

**DR-02 算出値の非冗長化**
未入金総額・遅延・推定納税額・予測残高などの集計/判定値はテーブルに保存せず、都度算出する（状態の不整合を防ぐため）。

**DR-03 主要エンティティ**

| エンティティ | 主な属性 | 関連 |
|---|---|---|
| invoices | status, invoice_number, invoice_date, due_date, paid_date, billed_amount, paid_amount, memo | projects と N:1 |
| tax_settings | 各税率, 控除, 消費税有効フラグ | 単一レコード |
| dashboard_cards | card_key, is_visible, sort_order | ― |
| cashflow_snapshots（任意） | balance_date, balance | ― |

**DR-04 既存エンティティへの影響**
- projects は invoices と 1:N。
- fixed_costs / subscriptions はキャッシュフロー算出に使用（支払日属性が必要な場合は追加を検討）。
- 売上は請求ベース、キャッシュフローは入金ベースで区別して集計する。

## 6. 非機能要件

**NFR-01 パフォーマンス**
主要画面（ダッシュボード）は高速に表示され、毎日ストレスなく開ける応答性を持つこと。

**NFR-02 保守性**
カードとAPIは疎結合とし、新カード追加が「レジストリへの1件追加」で完結する構造にすること。

**NFR-03 拡張時のプロセス**
新機能追加時は、プロダクト原則4項目の適合確認と、DB/API/UIへの影響整理を事前に文書化すること。

**NFR-04 シンプルさ**
必要以上に機能を増やさない。過剰な権限管理・監査ログ・汎用会計機能は実装しない。

**NFR-05 単一ユーザー運用**
自分専用を前提とし、認証は単一ユーザー（またはパスコード）で足りること。マルチテナントは不要。

---

## 7. 制約・前提

- 個人利用（単一ユーザー）を前提とする。
- 税額はあくまで概算であり、正式な納税額は確定申告で確定する。
- データ規模は個人利用相当であり、集計はリアルタイム算出で性能上の問題を生じない前提とする。

---

## 8. 対象外（Non-Goals）

- 汎用的な複式簿記・仕訳帳の作成
- 複数ユーザー / チーム / 権限管理
- 税務署提出用の正式な確定申告書生成
- 顧客請求書の送付・入金消込の外部連携（初期段階では対象外）
